자바스크립트 배열 이해하기#

index로 값 가져오기#

var array = [1, 2, 3, 4];
array[2] // return 3

값 할당하는 방법#

front#

자바스크립트에서 배열에 값을 추가적으로 할당하거나 변경하라

back#

var array = [1,2,3,4];
array[2] = "abcd"

길이#

array.length를 통해서 배열의 길이를 얻을 수 있다.

끝에 추가하기#

array.push(10을 통해서 값을 획득할 수 있다. 반환값은 추가된 마지막 배열의 위치 index이다.

var array = [1, 2, 3, 4];
array.push(10) // return 5
// array의 맨뒤에 5가 추가되서 [1,2,3,4,5] 가 된다.

끝에 있는 값 지우기#

var array = [1, 2, 3, 4];
array.pop();
// 4
// > array는 [1,2,3] 이 되고 리턴은 4가 된다.

앞에 추가하기#

var array = [1,2,3,4];
array.unshift(10);
// 새로운 길이인 5가 리턴된다.
// array는 [10, 1, 2, 3, 4]가 된다.

  • front

    자바스크립트의 배열에서 가장 앞에 값을 추가하라.

  • back

    unshift(eliment)를 사용하면 값이 추가된다.

앞에 있는 값 지우기#

var array = [1,2,3,4];
array.shift() // return 지운값
// array는 [2,3,4]가 된다.

Item#

  • front

    자바스크립트에서 배열 제일 앞에 있는 값을 지우는 함수는 무엇인가?

  • back

    shift()를 사용하고 리턴은 지운값이 나오게 된다.

배열 얇게 복사하기#

slice(); 함수를 사용한다.

항목 삭제 하기#

var array = [1,2,3,4,5,6];
array.splice(2,3); // [3,4,5]
array // [1,2,6]

splice를 사용한다.

Item#

  • front

    자바스크립트 배열에서 항목을 삭제하는 방법에 대해서 말하라.

  • back

    splice함수를 쓰면 Index, count를 입력해서 배열에서 값을 삭제할 수 있다.

    var array = [1,2,3,4,5,6];
array.splice(2,3); // [3,4,5]
array // [1,2,6]

위의 기본 함수들의 문제점#

Argument로 받은 값들의 Reference를 통해서 값이 직접 수정되고 있다는 것 자체가 문제이기 때문에, 카피온 라이트의 형태로 모든 함수를 변경하는 것이 더 좋다.

예를 들어서 shift와 같은 함수들은 다음과 같이 Copy On Write로 변경할 수 있다.

function safty_shift(arr) {
  var copy = arr.slice();
  copy.shift();
  return copy;
}

npm-package#

@angular/cli와 같은 At(@)이 포함된 package 이름은 왜 그럴까?#

:PROPERTIES: :ID: D67E112A-3596-4853-900D-686F5ED05836 :EXPORT_FILE_NAME: what-is-js-package-at.md :EXPORT_DATE: 2022-12-03 :ENDin:

When you sign up for an npm user account or create an Org, you are granted a scope that matches your user or Org name. You can use this scope as a namespace for related packages. A scope allows you to create a package with the same name as a package created by another user or Org without conflict.

당신이 NPM user 계정을 만들거나, org를 만들면 사용자 또는 org이름과 일치된 scope가 제공된다. 이 scope를 패키지의 네임스페이스로 활용할 수 있다. Scope를 사용하면 충돌 없이 다른 사용자 또는 조직에서 만든 패키지와 동일한 이름의 패키지를 만들 수 있다.
